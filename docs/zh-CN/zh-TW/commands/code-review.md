# 程序代码审查

对未提交变更进行全面的安全性和品质审查：

1. 取得变更的档案：git diff --name-only HEAD

2. 对每个变更的档案，检查：

**安全性问题（关键）：**
- 寫死的憑证、API 金钥、Token
- SQL 注入弱点
- XSS 弱点
- 缺少输入验证
- 不安全的相依性
- 路徑遍历风险

**程序代码品质（高）：**
- 函数 > 50 行
- 档案 > 800 行
- 巢状深度 > 4 层
- 缺少错误处理
- console.log 陈述式
- TODO/FIXME 注释
- 公开 API 缺少 JSDoc

**最佳实务（中）：**
- 变异模式（应使用不可变）
- 程序代码/注释中使用表情符号
- 新程序代码缺少测试
- 无障碍问题（a11y）

3. 产生報告，包含：
   - 嚴重性：关键、高、中、低
   - 档案位置和行号
   - 问题描述
   - 建议修复

4. 如果发现关键或高优先问题则阻擋提交

绝不批准有安全弱点的程序代码！
