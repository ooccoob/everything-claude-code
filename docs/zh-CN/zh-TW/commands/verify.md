# 验证指令

对目前程序代码库状态执行全面验证。

## 说明

按此确切顺序执行验证：

1. **构建检查**
   - 执行此专案的构建指令
   - 如果失敗，報告错误并停止

2. **型別检查**
   - 执行 TypeScript/型別检查器
   - 報告所有错误，包含 档案:行号

3. **Lint 检查**
   - 执行 linter
   - 報告警告和错误

4. **测试套件**
   - 执行所有测试
   - 報告通过/失敗数量
   - 報告覆盖率百分比

5. **Console.log 稽核**
   - 在原始档案中搜索 console.log
   - 報告位置

6. **Git 状态**
   - 显示未提交的变更
   - 显示上次提交后修改的档案

## 输出

产生简洁的验证報告：

```
验证：[通过/失敗]

构建：    [OK/失敗]
型別：    [OK/X 个错误]
Lint：    [OK/X 个问题]
测试：    [X/Y 通过，Z% 覆盖率]
密钥：    [OK/找到 X 个]
日志：    [OK/X 个 console.logs]

准备好建立 PR：[是/否]
```

如果有任何关键问题，列出它们并提供修复建议。

## 參数

$ARGUMENTS 可以是：
- `quick` - 只检查构建 + 型別
- `full` - 所有检查（预设）
- `pre-commit` - 与提交相关的检查
- `pre-pr` - 完整检查加上安全性掃描
