# /learn - 撷取可重用模式

分析目前的工作阶段并撷取值得保存为技能的模式。

## 触发

在工作阶段中任何时间点解决了非琐碎问题时执行 `/learn`。

## 撷取内容

尋找：

1. **错误解决模式**
   - 发生了什麼错误？
   - 根本原因是什麼？
   - 什麼修复了它？
   - 这可以重用於类似错误嗎？

2. **调试技术**
   - 非显而易见的调试步骤
   - 有效的工具组合
   - 诊斷模式

3. **变通方案**
   - 函数库怪癖
   - API 限制
   - 特定版本的修复

4. **专案特定模式**
   - 发现的程序代码库慣例
   - 做出的架构决策
   - 整合模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 建立技能档案：

```markdown
# [描述性模式名称]

**撷取日期：** [日期]
**上下文：** [此模式何时适用的简短描述]

## 问题
[此模式解决什麼问题 - 要具体]

## 解决方案
[模式/技术/变通方案]

## 范例
[如适用的程序代码范例]

## 何时使用
[触发条件 - 什麼应该启动此技能]
```

## 流程

1. 审查工作阶段中可撷取的模式
2. 识别最有价值/可重用的见解
3. 起草技能档案
4. 请使用者在保存前确认
5. 保存到 `~/.claude/skills/learned/`

## 注意事项

- 不要撷取琐碎的修复（打字错误、简单的语法错误）
- 不要撷取一次性问题（特定 API 停机等）
- 专注於会在未来工作阶段节省时间的模式
- 保持技能专注 - 每个技能一个模式
