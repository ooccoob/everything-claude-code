# 测试需求

## 最低测试覆盖率：80%

测试类型（全部必要）：
1. **单元测试** - 个別函数、工具、元件
2. **整合测试** - API 端点、资料库操作
3. **E2E 测试** - 关键使用者流程（Playwright）

## 测试驱动开发

強制工作流程：
1. 先撰寫测试（RED）
2. 执行测试 - 应该失敗
3. 撰寫最小实作（GREEN）
4. 执行测试 - 应该通过
5. 重构（IMPROVE）
6. 验证覆盖率（80%+）

## 测试失敗疑难排解

1. 使用 **tdd-guide** Agent
2. 检查测试隔离
3. 验证 mock 是否正确
4. 修复实作，而非测试（除非测试是错的）

## Agent 支援

- **tdd-guide** - 主动用於新功能，強制先撰寫测试
- **e2e-runner** - Playwright E2E 测试专家
